# Milestone v1.4.0: Agent Clarity

**Status:** ✅ SHIPPED 2026-03-16
**Phases:** 8-10
**Total Plans:** 7

## Overview

This milestone made the `Agents` workspace dramatically easier to scan and trust. It rebuilt the surface around explicit Working / Blocked / Idle triage, retired the old repo-task runtime split from live dashboard semantics, and then added provenance-aware workloads plus a concise coordination brief so operators can tell what agents are serving without opening each thread.

## Milestone Summary

**Key Decisions:**

- Keep the `Agents` surface roster-first and leave ownership truth in Mission Control.
- Treat `task-center`, `agent-launchpad`, and `agent-workflow` as archived context rather than live repo-task ownership systems.
- Expand idle suggestions using repo and research backlogs, but keep the results explicitly advisory and non-owning.
- Show provenance confidence directly in the UI so partial metadata never reads like exact truth.

**Issues Resolved:**

- Operators can now tell who is working, blocked, or idle within seconds of opening the Agents view.
- Mission Control no longer teaches or depends on the retired three-system repo-task split.
- Working agents now show explicit repo or intake provenance, and idle agents get clearer advisory next moves without reopening ownership ambiguity.
- The default lower-half Agents surface now prioritizes coordination instead of overwhelming analytics rails.

**Issues Deferred:**

- Mission Control task-to-agent mapping beyond the agent-focused provenance added here remains future work.
- Deeper overlap, handoff, and coordination analytics are still candidates for the next milestone.
- No new milestone is defined yet after shipping `1.4.0`.

**Technical Debt Incurred:**

- Nyquist validation artifact coverage is partial because Phases 8 and 9 do not have standalone `VALIDATION.md` files.
- Provenance precision still depends on the richness of local session metadata; some live desks may continue to show partial confidence markers.
