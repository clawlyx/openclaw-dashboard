# Milestone v1.2.0: Explicit Ownership Workstation

**Status:** ✅ SHIPPED 2026-03-14
**Phases:** 1-4
**Total Plans:** 8

## Overview

This milestone turned the joined workstation into an explicit ownership console. It made ownership a first-class snapshot contract, added focused room and agent drawers, reused Mission Control actions from the office surface, and finished with pressure signals plus a public-safe `1.2.0` release package.

## Phases

### Phase 1: Ownership Contract

**Goal**: Make explicit mission ownership and structured task references available to the dashboard without breaking existing fallback behavior.  
**Depends on**: Nothing (first phase)  
**Plans**: 2 plans

Plans:

- [x] 01-01: Extend mission and agent snapshot types, readers, and demo data for explicit ownership fields
- [x] 01-02: Join explicit ownership into dashboard presenters and label inferred fallback in the office surface

**Details:**
- Exposed `ownerAgentId`, `ownerRoomId`, `startedAt`, `lastWorkedAt`, `nextPlannedStep`, `blockedReason`, and `waitingOn` in Mission Control task snapshots.
- Added `currentTaskId`, `lastTaskId`, and `nextTaskId` to agent snapshots.
- Preserved readable inferred fallback when live installs do not emit ownership fields yet.

### Phase 2: Detail Drawers

**Goal**: Add focused room and agent detail drawers that make current ownership and recent handoff context inspectable from the office view.  
**Depends on**: Phase 1  
**Plans**: 2 plans

Plans:

- [x] 02-01: Build drawer state, shell, and room/agent navigation wiring
- [x] 02-02: Populate drawer content with ownership details, handoff context, and artifact links

**Details:**
- Added shared room and agent drawer routing from room cards, mission cards, and desk cards.
- Surfaced current task, previous task, next step, blockers or waiting state, recent handoffs, and artifact links without leaving the office surface.

### Phase 3: Office Actions

**Goal**: Let the operator run supported Mission Control transitions from the office surface without losing server truth.  
**Depends on**: Phase 2  
**Plans**: 2 plans

Plans:

- [x] 03-01: Wire supported task mutations into office drawer and queue actions
- [x] 03-02: Refresh state from Mission Control truth and verify local and remote action flows

**Details:**
- Reused the Mission Control action catalog and task mutation path in the office drawer.
- Kept unsupported transitions visible but disabled.
- Repaired post-mutation focus so the office drawer follows refreshed ownership after task transitions.

### Phase 4: Pressure Signals

**Goal**: Prioritize urgent work and package the milestone with refreshed demo data, docs, and acceptance coverage.  
**Depends on**: Phase 3  
**Plans**: 2 plans

Plans:

- [x] 04-01: Add pressure heuristics, scoring, and attention ordering to the office surface
- [x] 04-02: Refresh demo data, docs, screenshots, and release validation for `1.2.0`

**Details:**
- Added a reusable pressure model for stale review, blocked-too-long, waiting-on-human, inferred ownership, and room overload.

- Refreshed bundled demo state, README docs, release runbook, and GitHub homepage screenshots from public-safe demo data only.

## Milestone Summary

**Key Decisions:**

- Keep `openclaw-dashboard` as the primary operator app and absorb user-facing Mission Control context into one workstation.
- Treat Mission Control as the delivery source of truth while the office surface handles visualization and supported transitions.
- Keep ownership fallback visible instead of pretending inferred routing is exact truth.
- Derive pressure from explainable snapshot data instead of hidden scoring heuristics.

**Issues Resolved:**

- Mission ownership no longer depends only on room or lane inference.
- Operators no longer need to switch away from the office view to inspect artifacts or run common task transitions.
- Public screenshots now use a truly safe demo path for both OpenClaw and Mission Control data.

**Issues Deferred:**

- No standalone milestone-audit artifact was produced before archival; completion is based on completed phase summaries plus final verification evidence.
- Broader operator intelligence metrics and host-runtime controls remain future work.

**Technical Debt Incurred:**

- Pressure scoring is still purely current-state based; historical SLA trends and aging analytics remain for a later milestone.
- Mission Control sample data lives in code and will need periodic refresh as the workstation evolves.
