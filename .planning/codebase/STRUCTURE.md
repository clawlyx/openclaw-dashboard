## Directory Purposes

**app/**
- Purpose: Own the Next.js App Router shell, route-level metadata, global CSS, and HTTP endpoints.
- Contains: `page.tsx`, `layout.tsx`, `globals.css`, `icon.svg`, and `route.ts` files under `app/api/...`.
- Key files: `app/page.tsx`, `app/layout.tsx`, `app/api/snapshot/route.ts`, `app/api/theme/route.ts`, `app/api/mission-control/intake/route.ts`, `app/api/mission-control/tasks/[taskId]/route.ts`.
- Subdirectories: `app/api/snapshot`, `app/api/theme`, `app/api/mission-control/intake`, `app/api/mission-control/tasks/[taskId]`.

**components/**
- Purpose: Hold reusable dashboard UI and the small set of client-side interactive components.
- Contains: Kebab-case `.tsx` component modules.
- Key files: `components/agents-office-panel.tsx`, `components/agents-virtual-office-panel.tsx`, `components/mission-control-panel.tsx`, `components/mission-intake-form.tsx`, `components/mission-task-actions.tsx`, `components/theme-switch.tsx`.
- Subdirectories: None; the directory is currently flat.

**lib/**
- Purpose: Concentrate non-UI logic, typed snapshot contracts, data-source readers, mutation helpers, and display-formatting helpers.
- Contains: Kebab-case `.ts` modules with domain-oriented exports.
- Key files: `lib/openclaw.ts`, `lib/agents.ts`, `lib/mission-control.ts`, `lib/mission-control-mutations.ts`, `lib/mission-control-actions.ts`, `lib/dashboard-presenters.ts`, `lib/i18n.ts`, `lib/theme.ts`.
- Subdirectories: None; all modules currently live at the top level.

**demo/openclaw-home/**
- Purpose: Provide a publishable fallback dataset so the dashboard can run without access to a real local OpenClaw install.
- Contains: Demo agent config, auth profiles, cron data, and usage markdown reports.
- Key files: `demo/openclaw-home/agents/dashboard.json`, `demo/openclaw-home/agents/main/agent/auth-profiles.json`, `demo/openclaw-home/cron/jobs.json`, `demo/openclaw-home/workspace/memory/usage/2026-03-01.md`.
- Subdirectories: `agents/`, `cron/`, `workspace/memory/usage/`.

**docs/**
- Purpose: Store product and delivery artifacts, including files generated by local mission-control mutations.
- Contains: Markdown docs for briefs, RFCs, QA plans, release packets, and a PRD.
- Key files: `docs/plan/openclaw-dashboard-prd.md`, `docs/briefs/F-0016-openclaw-dashboard.md`, `docs/rfc/F-0018-openclaw-dashboard-i18n-support.md`, `docs/qa/F-0022-openclaw-dashboard-light-dark-mode.md`, `docs/release/public-release-checklist.md`.
- Subdirectories: `docs/briefs`, `docs/plan`, `docs/qa`, `docs/release`, `docs/rfc`.

**locales/**
- Purpose: Centralize user-facing copy for supported languages.
- Contains: JSON locale dictionaries.
- Key files: `locales/en.json`, `locales/zh.json`.
- Subdirectories: None.

**scripts/**
- Purpose: Hold small repo-specific helpers used by npm scripts.
- Contains: Node-based utility scripts.
- Key files: `scripts/next-runner.mjs`.
- Subdirectories: None.

**.github/**
- Purpose: Keep GitHub automation and public-facing repo assets together.
- Contains: Workflow YAML, PR template markdown, screenshots, GIFs, and social preview media.
- Key files: `.github/workflows/ci.yml`, `.github/workflows/commit-policy.yml`, `.github/PULL_REQUEST_TEMPLATE.md`, `.github/assets/readme-demo.png`.
- Subdirectories: `.github/workflows`, `.github/assets`.

**output/playwright/**
- Purpose: Preserve visual verification artifacts from UI passes and README/screenshot work.
- Contains: PNG screenshots, render captures, and frame sequences.
- Key files: `output/playwright/dashboard-desktop.png`, `output/playwright/dashboard-mobile.png`, `output/playwright/frames/01-overview.png`.
- Subdirectories: `output/playwright/frames`, `output/playwright/render`.

**plans/ and runbooks/**
- Purpose: Keep implementation planning and ship-check guidance outside the app runtime.
- Contains: Markdown plans and release/E2E checklists.
- Key files: `plans/openclaw-dashboard-v2-mission-control.md`, `runbooks/openclaw-dashboard-v2-mission-control-e2e.md`.
- Subdirectories: None under the current layout.

## Key File Locations

**Entry Points:**
- `app/page.tsx`: Main dashboard request handler and single-route workspace switchboard.
- `app/layout.tsx`: Root layout that applies fonts, global styles, and theme state.
- `app/api/snapshot/route.ts`: JSON snapshot endpoint for the assembled dashboard state.
- `app/api/theme/route.ts`: Theme cookie setter and redirect endpoint.
- `app/api/mission-control/intake/route.ts`: Mission intake write endpoint.
- `app/api/mission-control/tasks/[taskId]/route.ts`: Mission task transition endpoint.
- `scripts/next-runner.mjs`: Local `pnpm dev` / `pnpm start` runner.

**Configuration:**
- `package.json`: Scripts, dependency list, engine floor, and package manager pin.
- `next.config.ts`: Next.js features such as `typedRoutes`.
- `tsconfig.json`: Strict TypeScript settings and the `@/*` path alias.
- `eslint.config.mjs`: ESLint baseline using `eslint-config-next/core-web-vitals`.
- `.env.example`: `OPENCLAW_HOME` and `DASHBOARD_URL` examples for local setup.

**Core Logic:**
- `lib/openclaw.ts`: Dashboard snapshot aggregation, usage-report parsing, live quota lookup, and cron ingestion.
- `lib/agents.ts`: Agent office snapshot loading from `agents/dashboard.json` or derived session data.
- `lib/mission-control.ts`: Mission-control read model, demo fallback, queue/review derivation, and worker heartbeat handling.
- `lib/mission-control-mutations.ts`: Local Launchpad write path and docs bundle syncing.
- `lib/mission-control-actions.ts`: Allowed mission-task transitions for local and remote modes.
- `lib/dashboard-presenters.ts`: Formatting and translation helpers that sit between raw snapshots and UI copy.
- `lib/i18n.ts`: Locale resolution and message lookup.

**Testing / Validation:**
- `.github/workflows/ci.yml`: Automated install, typecheck, and build on GitHub Actions.
- `output/playwright/`: Committed screenshot evidence from manual visual verification.
- There is no dedicated `tests/` directory in the current repo; validation is script- and screenshot-driven.

**Documentation:**
- `README.md`: Main English usage and setup guide.
- `README.zh-CN.md`: Chinese user-facing guide.
- `CONTRIBUTING.md`: Repo workflow and validation expectations.
- `SECURITY.md`: Vulnerability handling guidance.
- `CHANGELOG.md`: Release history.
- `docs/`: Product and delivery artifacts.

## Naming Conventions

**Files:**
- `kebab-case.ts` / `kebab-case.tsx`: Default pattern for modules and components, for example `lib/dashboard-presenters.ts` and `components/provider-limit-windows.tsx`.
- `page.tsx`, `layout.tsx`, `route.ts`: Next.js file-convention entrypoints under `app/`.
- `[taskId]`: Next.js dynamic route segment naming under `app/api/mission-control/tasks/[taskId]`.
- `en.json`, `zh.json`: Locale dictionaries keyed by short language code.
- `UPPERCASE.md`: Root project docs such as `README.md`, `CHANGELOG.md`, `CONTRIBUTING.md`, and `SECURITY.md`.

**Directories:**
- Flat top-level source directories: `app`, `components`, `lib`, `locales`, `scripts`.
- Plural collection directories where the contents are many related artifacts: `components`, `locales`, `plans`, `runbooks`, `docs/briefs`, `docs/rfc`, `docs/qa`, `docs/release`.
- Domain-specific nested directories under `demo/openclaw-home/` mirror the external OpenClaw data layout.

**Special Patterns:**
- `F-####-<slug>.md`: Feature artifact files in `docs/briefs`, `docs/rfc`, `docs/qa`, and `docs/release`.
- `YYYY-MM-DD.md`: Usage history reports under `demo/openclaw-home/workspace/memory/usage/`.
- Screenshot outputs use descriptive kebab-case filenames under `output/playwright/`.

## Where to Add New Code

**New Dashboard View or Panel:**
- Primary code: `app/page.tsx` for view/panel registration and panel routing.
- Extracted UI: `components/` for any reusable or large panel component.
- Copy: `locales/en.json` and `locales/zh.json`.

**New Read-Only Data Source or Snapshot Section:**
- Primary code: `lib/openclaw.ts` if it belongs in the unified dashboard snapshot, or a new `lib/<domain>.ts` module if it is a new domain.
- UI wiring: `app/page.tsx` plus a matching component in `components/`.
- Config if needed: `.env.example` and `README.md`.

**New Mission-Control Mutation or Workflow Rule:**
- API boundary: `app/api/mission-control/...`.
- Transition rules: `lib/mission-control-actions.ts`.
- Local persistence: `lib/mission-control-mutations.ts`.
- Generated artifact handling: `docs/briefs`, `docs/rfc`, `docs/qa`, `docs/release`.

**New Shared Utility or Formatter:**
- Implementation: `lib/`.
- If it is purely presentational, prefer `components/` only when it renders JSX.

**New Validation Coverage:**
- Automated pipeline: extend `.github/workflows/ci.yml`.
- Visual/manual evidence: add or refresh captures under `output/playwright/`.
- There is no current in-repo unit test folder to plug into.

## Special Directories

- Purpose: Bundled demo data used when no local OpenClaw home is available.
- Source: Committed fixtures maintained inside this repo.
- Committed: Yes.

- Purpose: Feature delivery artifacts, including files written by `lib/mission-control-mutations.ts` in local mode.
- Source: Manual planning work plus local mission-control sync writes.
- Committed: Yes.

**output/playwright/**
- Purpose: Generated screenshots and render passes for UI validation and public preview assets.
- Source: Local browser verification runs.
- Committed: Yes.

**.planning/codebase/**
- Purpose: GSD-generated mapping docs such as this file and `ARCHITECTURE.md`.
- Source: Brownfield codebase mapping work.
- Committed: Intended to be committed when mapping is refreshed.

**public/**
- Purpose: Static asset bucket for future public files.
- Source: Manual additions only.
- Committed: Yes, but currently empty.
